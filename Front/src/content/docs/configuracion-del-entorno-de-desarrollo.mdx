## Propósito y Alcance

<p>Este documento proporciona una guía completa para configurar un entorno de desarrollo local para el sistema de microservicios FitLive. Cubre la configuración del IDE, la configuración de la herramienta de compilación, la configuración del control de versiones y la preparación de la base de datos de desarrollo.</p>
<p>Para obtener información sobre cómo compilar y ejecutar los servicios después de la configuración, consulte <a href="/docs/primeros-pasos" class="content-link">Primeros Pasos</a>. Para el despliegue en producción y la configuración específica del entorno, consulte <a href="/docs/servicio-clientes/configuracion-y-despliegue" class="content-link">Configuración y Despliegue de servicio-clientes</a> y <a href="/docs/servicio-facturacion/configuracion-y-ajuste" class="content-link">Configuración y Ajuste de servicio-facturacion</a>.</p>

## Configuración del Maven Wrapper

<p>Los servicios de FitLive utilizan el <strong>Maven Wrapper</strong> (<code class="artifact-id">mvnw</code>), que empaqueta una versión específica de Maven con cada servicio. Esto elimina la necesidad de que los desarrolladores instalen Maven globalmente y asegura compilaciones consistentes en todos los entornos.</p>

<ol>

## Configuración de la Base de Datos de Desarrollo

<p>Los servicios de FitLive soportan tanto <strong>H2</strong> (en memoria) para desarrollo como <strong>PostgreSQL</strong> para producción.</p>

### Base de Datos H2 en Memoria (Por Defecto para Desarrollo)

<p>Los servicios están preconfigurados para usar la base de datos H2 para el desarrollo local, sin requerir la instalación de una base de datos externa.</p>

<ImageViewer 
    src="/src/assets/configuracion-bd-h2.svg" 
    alt="Diagrama que muestra cómo la aplicación Spring Boot, configurada a través de propiedades, se conecta a una base de datos H2 en memoria y proporciona acceso a través de la consola H2."
/>

<p><strong>Beneficios de la Configuración H2:</strong></p>
<ul>
    <li>No se requiere instalación de base de datos externa</li>
    <li>Arranque y cierre rápidos</li>
    <li>Creación automática del esquema a través de migraciones de Flyway</li>
    <li>Consola web integrada para la inspección de la base de datos</li>
</ul>
<p><strong>Acceso a la Consola H2:</strong></p>
<ul>
    <li>URL: <code>{"http://localhost:8080/h2-console"}</code></li>
    <li>JDBC URL: <code>jdbc:h2:mem:testdb</code></li>
    <li>Username: <code>sa</code></li>
    <li>Password: (vacío)</li>
</ul>

### Configuración de PostgreSQL (Desarrollo tipo Producción)

<p>Para los desarrolladores que necesiten probar contra PostgreSQL:</p>
<ol>
    <li>
        <strong>Instalar PostgreSQL:</strong>
        <CodeBlock language="bash" code={`# macOS (Homebrew)
brew install postgresql@14

# Ubuntu/Debian
sudo apt-get install postgresql-14

# Windows
# Descargar desde https://www.postgresql.org/download/windows/`} />
    </li>
    <li>
        <strong>Crear Base de Datos de Desarrollo:</strong>
        <CodeBlock language="sql" code={`CREATE DATABASE facturacion_db;
CREATE USER fitlive_user WITH ENCRYPTED PASSWORD 'dev_password';
GRANT ALL PRIVILEGES ON DATABASE facturacion_db TO fitlive_user;`} />
    </li>
    <li>
        <strong>Configurar Perfil de Aplicación:</strong>
        <ul>
            <li>Cree <code>application-postgres.properties</code> en <code>src/main/resources/</code></li>
            <li>O use variables de entorno para sobrescribir los valores por defecto</li>
        </ul>
    </li>
    <li>
        <strong>Exclusión de Archivos de Base de Datos:</strong>
        <ul>
            <SourceReference 
                variant="table" 
                label='' 
                textBefore="<li> El directorio <code>DB/</code> está excluido de Git vía "
                textAfter='</li>'

            <li>Los archivos de datos locales de PostgreSQL deben almacenarse fuera del repositorio</li>

    </li>
</ol>

## Solución de Problemas Comunes de Configuración

### Problemas de Permisos del Maven Wrapper (macOS/Linux)

<p><strong>Síntoma:</strong> <code class="artifact-id">Permission denied</code> al ejecutar <code class="artifact-id">./mvnw</code></p>
<p><strong>Solución:</strong></p>
<CodeBlock language="bash" code={`chmod +x mvnw
./mvnw --version`} />

    <li>Vuelva a clonar el repositorio con el manejo de finales de línea adecuado.</li>

### El IDE No Reconoce Java 22

<p><strong>Síntoma:</strong> El IDE muestra errores para sintaxis moderna de Java.</p>
<p><strong>Solución:</strong></p>
<ul>
    <li>Verifique que el JDK de Java 22 esté instalado: <code class="artifact-id">java -version</code></li>
    <li>Configure el SDK del proyecto en el IDE a Java 22.</li>
    <li>Actualice el IDE a la última versión que soporte Java 22.</li>
</ul>

### La Consola H2 No es Accesible

<p><strong>Síntoma:</strong> No se puede acceder a <code class="artifact-id">{"http://localhost:8080/h2-console"}</code></p>
<p><strong>Solución:</strong></p>
<ul>
    <li>Asegúrese de que la aplicación esté en ejecución: <code class="artifact-id">./mvnw spring-boot:run</code></li>
    <li>Verifique que la consola H2 esté habilitada en <code class="artifact-id">application.properties</code>.</li>
    <li>Verifique que el puerto 8080 no esté en uso por otra aplicación.</li>
</ul>

### Fallos de Conexión a PostgreSQL

<p><strong>Síntoma:</strong> La aplicación no se puede conectar a PostgreSQL.</p>
<p><strong>Solución:</strong></p>
<ul>
    <li>Verifique que el servicio de PostgreSQL esté en ejecución.</li>
    <li>Revise las credenciales de la base de datos en la configuración.</li>
    <li>Asegúrese de que la base de datos <code class="artifact-id">facturacion_db</code> exista.</li>
    <li>Verifique que la red/firewall permita la conexión al puerto de PostgreSQL (por defecto 5432).</li>
</ul>
