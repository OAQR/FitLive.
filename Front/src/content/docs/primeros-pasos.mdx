<p>Esta página proporciona instrucciones para compilar, ejecutar y desarrollar los microservicios de FitLive localmente usando el Maven Wrapper. El sistema FitLive consiste en dos servicios Spring Boot: <code class="artifact-id">servicio-clientes</code> (gestión de clientes) y <code class="artifact-id">servicio-facturacion</code> (generación de facturas).</p>
<p>Para información sobre la arquitectura general del sistema y cómo interactúan los servicios, consulte <a href="/docs/arquitectura-del-sistema" class="content-link">Arquitectura del Sistema</a>. Para detalles de configuración y despliegue específicos de cada servicio, vea <a href="/docs/servicio-clientes/configuracion-y-despliegue" class="content-link">Configuración y Despliegue de servicio-clientes</a> y <a href="/docs/servicio-facturacion/configuracion-y-ajuste" class="content-link">Configuración e Instalación de servicio-facturacion</a>.</p>

## Resumen del Maven Wrapper

<p>El Maven Wrapper (<code class="artifact-id">mvnw</code> y <code class="artifact-id">mvnw.cmd</code>) es un script de shell y un archivo batch que asegura que todos los desarrolladores usen la misma versión de Maven sin requerir una instalación local. Ambos servicios usan la versión 3.3.4 del Wrapper, que descarga Apache Maven 3.9.11.</p>

## Construyendo los Servicios

### Clonar el Repositorio
<CodeBlock code={`git clone https://github.com/OAQR/FitLive.git
cd Fitlive`} />

### Construir servicio-clientes
<p>Navegue al directorio de <code class="artifact-id">servicio-clientes</code> y ejecute el Maven Wrapper:</p>

<p><strong>En Unix/Linux/macOS:</strong></p>
<CodeBlock code={`cd Back/servicio-clientes
./mvnw clean install`} />

<p><strong>En Windows:</strong></p>
<CodeBlock code={`cd Back\\servicio-clientes
mvnw.cmd clean install`} />

### Construir servicio-facturacion
<p>De manera similar, navegue al directorio de <code class="artifact-id">servicio-facturacion</code>:</p>

<p><strong>En Unix/Linux/macOS:</strong></p>
<CodeBlock code={`cd Back/servicio-facturacion
./mvnw clean install`} />

<p><strong>En Windows:</strong></p>
<CodeBlock code={`cd Back\\servicio-facturacion
mvnw.cmd clean install`} />

## Ejecutando los Servicios

### Ejecutar servicio-clientes
<p>Después de compilar, puede ejecutar el servicio usando Maven:</p>
<CodeBlock code={`cd Back/servicio-clientes
./mvnw spring-boot:run`} />
<p>El servicio se inicia en el puerto <strong>8080</strong> por defecto y expone los endpoints de la API REST en <code>{"http://localhost:8080/api/v1/clientes"}</code>.</p>

### Ejecutar servicio-facturacion
<CodeBlock code={`cd Back/servicio-facturacion
./mvnw spring-boot:run`} />
<p>El servicio se inicia en su puerto configurado y proporciona capacidades de generación de facturas.</p>

### Alternativa: Ejecutar el JAR Empaquetado
<p>Después de ejecutar <code>mvnw package</code>, puede ejecutar el JAR independiente:</p>
<CodeBlock code={`java -jar target/servicio-clientes-<version>.jar`} />

## Tareas Comunes de Desarrollo

### Ejecutar Solo Pruebas
<p>Ejecute todas las pruebas sin empaquetar:</p>
<CodeBlock code={`./mvnw test`} />

### Omitir Pruebas Durante la Compilación
<p>Compile y empaquete sin ejecutar pruebas (para compilaciones más rápidas):</p>
<CodeBlock code={`./mvnw clean install -DskipTests`} />

### Compilación Limpia
<p>Elimine todos los artefactos de compilación anteriores y recompile desde cero:</p>
<CodeBlock code={`./mvnw clean package`} />

### Ver Dependencias
<p>Liste todas las dependencias del proyecto y sus versiones:</p>
<CodeBlock code={`./mvnw dependency:tree`} />

### Ejecutar en Modo de Depuración
<p>Inicie el servicio con la depuración remota habilitada en el puerto 5005:</p>
<CodeBlock code={`./mvnw spring-boot:run -Dspring-boot.run.jvmArguments="-Xdebug -Xrunjdwp:transport=dt_socket,server=y,suspend=n,address=5005"`} />

## Configuración del Entorno

### Perfil de Desarrollo
<p>Ambos servicios usan una base de datos en memoria H2 por defecto para el desarrollo local. No se requiere configuración de base de datos externa.</p>

### Perfil de Producción
<p>Para ejecutar con PostgreSQL (configuración de producción), establezca el perfil de Spring:</p>
<CodeBlock code={`./mvnw spring-boot:run -Dspring-boot.run.profiles=prod`} />
<p>O al ejecutar el JAR:</p>
<CodeBlock code={`java -jar target/servicio-clientes-<version>.jar --spring.profiles.active=prod`} />
<p>Para más opciones de configuración, consulte <a href="docs/servicio-cliente/configuracion-y-despliegue" class="content-link">Configuración y Despliegue de servicio-clientes</a> y <a href="/docs/servicio-facturacion/configuracion-y-ajuste" class="content-link">Configuración e Instalación de servicio-facturacion</a>.</p>

## Solución de Problemas

### Problemas de Descarga del Maven Wrapper
<p>Si el Maven Wrapper falla al descargar Maven automáticamente:</p>
<p><strong>Verifique la conectividad de red:</strong></p>
<CodeBlock code={`curl -I https://repo.maven.apache.org/maven2/org/apache/maven/apache-maven/3.9.11/apache-maven-3.9.11-bin.zip`} />

<p><strong>Configuración manual de MAVEN_HOME:</strong> Establezca la variable de entorno <code>MAVEN_HOME</code> para omitir la descarga automática:</p>
<CodeBlock code={`export MAVEN_HOME=/path/to/your/maven
./mvnw clean install`} />

<p><strong>Use un repositorio corporativo:</strong> Establezca <code>MVNW_REPOURL</code> para usar un repositorio de Maven corporativo:</p>
<CodeBlock code={`export MVNW_REPOURL=https://your-corporate-repo/maven2
./mvnw clean install`} />

### Desajuste de Versión de Java
<p>Los servicios requieren Java 22. Verifique su versión de Java:</p>
<CodeBlock code={`java -version`} />
<p>Si usa una versión diferente, establezca <code>JAVA_HOME</code>:</p>
<CodeBlock code={`export JAVA_HOME=/path/to/jdk-22
./mvnw clean install`} />

### Puerto Ya en Uso
<p>Si el puerto 8080 ya está en uso al iniciar <code class="artifact-id">servicio-clientes</code>:</p>
<p><strong>Opción 1: Cambiar el puerto en la línea de comandos:</strong></p>
<CodeBlock code={`./mvnw spring-boot:run -Dspring-boot.run.arguments=--server.port=8081`} />
<p><strong>Opción 2: Terminar el proceso que usa el puerto 8080:</strong></p>
<CodeBlock code={`# En Unix/Linux/macOS
lsof -ti:8080 | xargs kill -9

# En Windows
netstat -ano | findstr :8080
taskkill /PID <PID> /F`} />
